Blakenwite by Timothy Wojtylak

A greyscale plugin

Introduction

Blakenwite started out as a project to get me used to the ins and outs of creating a plugin while helping me understand github and version control. There were a few obstacles along the way, which you'll read about later, but nonetheless, it has been a great learning experience and I hope to only make it better.

With that being said, let's get started...

Parameter Breakdown

As of right now, there exists only two parameters, 'dull' and 'bright.'

Parameter Description
Dull Sends all child and descendant elements through the greyscale function
and sets their properties accordingly.

$('#someElement').bw('dull');
Bright Returns all child and descendant elements to their original colors.

$('#someElement').bw('bright');

Setup

The process for setting up Blakenwite couldn't be any easier. The first thing you'll need to do is get your hands on the plugin. You can download the zip from github and extract it into your development environment.

Download from Github

Once you have that extracted, you'll notice all the files are in one folder called "BW." There is a minified version coming in at a whopping 1080 bytes and the unminified version bringing up the rear at 1960 bytes.

The next step is including the file into your webpage. Blakenwite depends on jQuery in order to work and if you include jQuery AFTER you've included Blakenwite, you won't get the results you were expecting.

Limitations

There are a few limitations you should be aware of while utilizing Blakenwite:

  1. Images are not yet supported. This means all your images will remain in color. Now, I have a working version that can greyscale all images, however, it is slow and not very efficient. So for now, no images.
  2. Dynamically added content is not supported, yet. If you have content that is dynamically loaded into your webpage, it will turn grey, but it won't come back from it. The reason for this is explained below in the 'How It Works' section.
  3. Gradients aren't supported right now.

How It Works

Once the document has fully loaded, Blakenwite creates a virtual tree of your DOM and the styles of each element. This is so it can go back to the original color when using the 'bright' parameter.

When the parameter 'dull' is passed, Blakenwite takes the element being invoked upon and finds all its children and descendants. It takes what the colors are for their various properties and sends them through the greyscale algorithm to determine what shade of grey matches the set of RGB values for each property. Finally, it sets the colors accordingly onto each element being affected.

To go from the 'dull' state to the 'bright' state, the element being invoked upon is found within the virtual DOM tree and those initial color values for the children and descendants of that element are reset.
